## 项目概述

**Manim Video API Service** 是一个基于 LangGraph 的多智能体视频生成 API 服务，能够将结构化的教学需求转换为 Manim 代码，自动渲染视频，智能修复错误，并将最终产物上传至阿里云 OSS。

### 主要特点

- 🤖 **多智能体协作**：使用 Planner、Code Expert、Diagnoser、Repair 等多个专业 Agent 协同工作
- 🔄 **智能错误修复**：自动诊断渲染失败原因并进行代码修复，最多支持 6 次重试
- 📊 **LangGraph 编排**：基于状态图的工作流引擎，实现复杂的条件分支和循环逻辑
- ☁️ **云端存储**：支持阿里云 OSS 自动上传和管理生成的视频文件
- 🎯 **精准时序控制**：支持音频、字幕、动画的精确时间轴对齐
- 🔍 **完整可观测性**：详细的日志记录和任务状态追踪

## 核心功能

### 1. 智能代码生成
- 根据教学大纲和场景描述自动生成 Manim 代码
- 支持多种布局模板（居中聚焦、左右分栏、上下结构等）
- 自动处理中文字体、配色方案和视觉样式

### 2. 自动渲染与修复
- 调用 Manim Community 引擎渲染视频
- 渲染失败时自动诊断错误类型
- 智能修复代码问题并重试（最多 6 次）
- 保存所有尝试的代码版本和日志

### 3. 音视频合成
- 支持场景级音频文件集成
- 自动生成时间轴对齐的字幕
- 可选背景音乐和音量调节
- 音频标准化和静音插入

### 4. 云存储管理
- 自动上传视频到阿里云 OSS
- 生成可公开访问的视频链接
- 支持自定义存储路径前缀
- 可选 STS 临时令牌认证

### 5. 任务状态追踪
- 实时查询任务执行状态
- 查看历史尝试记录和诊断信息
- 下载中间产物和调试文件

### 5. 启动服务

#### ⚠️ Windows 用户特别注意（CMD vs PowerShell）

在 Windows 环境下，不同的命令行工具对参数的解析方式不同，请根据你的环境选择正确的启动方式：

**方案 A：使用推荐脚本（最稳妥）**
项目已提供配置好的启动脚本，自动处理了路径和参数兼容性问题：
```powershell
# PowerShell 中运行
.\scripts\dev\start_dev.ps1
```

**方案 B：手动启动 - PowerShell 环境**
如果你习惯使用 PowerShell，请注意**不要复制命令提示符**（如 `( .venv ) D:\...>`），且通配符解析与 CMD 不同：
```powershell
# 激活环境后运行
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```
*注：PowerShell 下 `--reload-exclude "runtime/*"` 有时会失效或报错，建议手动启动时先省略该参数。*

**方案 C：手动启动 - CMD (命令提示符) 环境**
在 CMD 中，`*` 通配符会被系统展开成文件名，导致 `uvicorn` 报“额外参数”错误。
**错误示范**：`--reload-exclude runtime/*` (会报错: Got unexpected extra argument)
**正确做法**：针对具体后缀进行排除，或直接省略：
```cmd
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

## 配置说明

### 环境变量详解

编辑 `.env` 文件进行配置：

#### 应用基础配置

```ini
# 应用名称（用于日志标识）
APP_NAME=Manim Video API Service

# 运行环境：dev / staging / prod
APP_ENV=dev

# 监听地址（0.0.0.0 允许外部访问）
HOST=0.0.0.0

# 监听端口
PORT=8000

# 日志级别：DEBUG / INFO / WARNING / ERROR / CRITICAL
LOG_LEVEL=INFO
```

#### 运行时配置

```ini
# 任务运行时根目录（存储所有任务的执行文件、日志和产物）
RUNTIME_ROOT=runtime/tasks

# Prompt 模板根目录
PROMPT_ROOT=prompts

# 最大重试次数（渲染失败时的自动修复尝试上限）
MAX_ATTEMPTS=6
```

#### LLM 配置

```ini
# 是否启用 LLM 辅助功能（代码生成、错误诊断等）
ENABLE_LLM_ASSIST=false

# LLM 提供商：mock / openai / azure / custom
LLM_PROVIDER=mock

# LLM API 基础 URL（OpenAI 兼容接口）
# 示例：https://api.openai.com/v1
LLM_BASE_URL=

# LLM API 密钥
LLM_API_KEY=

# LLM 模型名称
# 示例：gpt-4, gpt-3.5-turbo, claude-3-opus
LLM_MODEL=
```

**LLM 配置示例（使用 OpenAI）：**

```ini
ENABLE_LLM_ASSIST=true

LLM_BASE_URL=https://api.openai.com/v1
```

## API 接口

### 1. 视频渲染

**端点**: `POST /v1/video/render`

**功能**: 提交视频渲染请求，同步返回任务结果

**请求体**: `RenderRequest`

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| requestId | string | 否 | 请求唯一标识（自动生成） |
| projectBrief | ProjectBrief | 是 | 项目概要信息 |
| timedScenes | TimedScene[] | 是 | 带时序的场景列表 |
| renderPolicy | RenderPolicy | 否 | 渲染策略配置 |
| outputPolicy | OutputPolicy | 否 | 输出策略配置 |
| storagePolicy | StoragePolicy | 否 | 存储策略配置 |
| audioPolicy | AudioPolicy | 否 | 音频策略配置 |
| bgmPolicy | BgmPolicy | 否 | 背景音乐策略 |
| debugOptions | DebugOptions | 否 | 调试选项 |

**响应体**: `RenderResponse`

| 字段 | 类型 | 说明 |
|------|------|------|
| taskId | string | 任务唯一标识 |
| success | boolean | 是否成功 |
| status | string | 任务状态（success/failed） |
| videoUrl | string | 视频访问 URL（OSS 启用时） |
| ossObjectKey | string | OSS 对象键 |
| message | string | 消息（失败时包含错误信息） |
| attempts | integer | 尝试次数 |
| taskDir | string | 任务工作目录路径 |

**示例**:

```bash
curl -X POST http://localhost:8000/v1/video/render \
  -H "Content-Type: application/json" \
  -d @sample_request.json
```

### 2. 查询任务状态

**端点**: `GET /v1/video/tasks/{task_id}`

**功能**: 查询指定任务的详细信息和历史记录

**路径参数**:

| 参数 | 类型 | 说明 |
|------|------|------|
| task_id | string | 任务 ID |

**响应体**: `TaskEnvelope`

| 字段 | 类型 | 说明 |
|------|------|------|
| task | object | 任务状态详情 |
| result | object | 最终结果（如果已完成） |

**示例**:

```bash
curl http://localhost:8000/v1/video/tasks/task-abc123
```

### 3. 健康检查

**端点**: `GET /health`

**功能**: 检查服务健康状态

**响应**:

```json
{
  "status": "ok",
  "service": "Manim Video API Service"
}
```

### 各阶段详细说明

#### 1. 任务初始化 (`initialize_task`)
- 生成唯一任务 ID
- 创建任务工作目录结构
- 初始化任务状态文件

#### 2. 需求规划 (`plan_request`)
- 解析项目概要
- 分析场景复杂度
- 制定代码生成策略
- 选择合适的布局模板

#### 3. 加载通知 (`load_notices`)
- 读取历史验证通过的通知
- 读取历史被拒绝的通知
- 提取学习经验避免重复错误

#### 4. 代码生成 (`generate_code`)
- **方式选择**：根据配置选择 LLM 生成或直接生成
- **LLM 模式**：
  - 调用 Code Expert Agent
  - 结合 Prompt 模板和编码规则
  - 生成符合规范的 Manim 代码
- **直接模式**：
  - 使用确定性模板
  - 适用于简单场景

#### 5. 渲染执行 (`render_code`)
- 调用 Manim 渲染引擎
- 设置渲染参数（分辨率、FPS、背景色）
- 捕获渲染日志和错误输出
- 生成视频文件

#### 6. 错误诊断 (`diagnose_errors`)
- 解析渲染日志
- 识别错误类型：
  - 语法错误
  - 导入错误
  - 运行时错误
  - Manim API 误用
- 定位错误行号和原因

#### 7. 代码修复 (`repair_code`)
- 根据诊断结果生成修复方案
- 修改问题代码
- 保持原有结构和风格
- 记录修复说明

#### 8. 视频上传 (`upload_video`)
- 校验视频文件完整性
- 生成 OSS 对象键
- 上传到阿里云 OSS
- 生成公共访问 URL

## 故障排查

### 常见问题

#### 1. 服务无法启动

**症状**: `uvicorn` 命令报错

**解决方案**:
```bash
# 检查 Python 版本
python --version  # 需要 ≥3.11

# 重新安装依赖
pip install -r requirements.txt

# 检查端口占用
netstat -ano | findstr :8000  # Windows
lsof -i :8000  # Linux/macOS
```

#### 2. Manim 渲染失败

**症状**: 任务状态为 `failed`，错误信息包含 Manim 相关错误

**排查步骤**:
```bash
# 检查 Manim 安装
manim --version

# 查看任务日志
cat runtime/tasks/{task_id}/logs/render.log

# 手动测试渲染
cd runtime/tasks/{task_id}/code
manim -pqh scene_file.py SceneName
```

**常见原因**:
- 缺少系统依赖（cairo、pango）
- 中文字体未安装
- Manim 版本不兼容

**解决方案**:
```bash
# Windows: 安装 MiKTeX 或 TeX Live
# Linux: 
sudo apt-get install texlive-full

# 安装中文字体
# Windows: 通常已预装 Microsoft YaHei
# Linux:
sudo apt-get install fonts-wqy-zenhei
```

#### 3. FFmpeg/FFprobe 找不到

**症状**: 错误信息包含 `ffprobe not found`

**解决方案**:
```bash
# 检查安装
ffprobe -version

# 配置路径
# 在 .env 中设置完整路径
FFPROBE_BIN=C:\ffmpeg\bin\ffprobe.exe
```

#### 4. OSS 上传失败

**症状**: 视频渲染成功但上传失败

**排查步骤**:
```bash
# 检查 OSS 配置
# 确认 .env 中的 OSS 凭证正确

# 测试网络连接
ping oss-cn-beijing.aliyuncs.com

# 查看上传日志
cat runtime/tasks/{task_id}/logs/upload.log
```

**常见原因**:
- AccessKey 无效或过期
- Bucket 不存在或无权限
- 网络不通

#### 5. LLM API 调用失败

**症状**: 代码生成阶段失败，日志显示 LLM 错误

**解决方案**:
```bash
# 检查 LLM 配置
# 确认 ENABLE_LLM_ASSIST=true
# 确认 LLM_API_KEY 正确

# 测试 API 连通性
curl https://api.openai.com/v1/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-4","messages":[{"role":"user","content":"test"}]}'

# 临时切换到 Mock 模式
ENABLE_LLM_ASSIST=false
LLM_PROVIDER=mock
```

#### 6. 内存不足

**症状**: 渲染过程中进程被杀死

**解决方案**:
```bash
# 降低分辨率
"videoSpec": {
  "resolution": "1280x720"
}

# 减少 FPS
"videoSpec": {
  "fps": 24
}

# 增加系统 swap
# Linux:
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

#### 7. 热重载频繁重启

**症状**: 服务不断重启

**解决方案**:
```bash
# 确保启动命令包含排除项
uvicorn app.main:app --reload --reload-dir app --reload-exclude "runtime/*"

# 或者使用提供的启动脚本
.\start_dev.ps1  # Windows
./start.sh       # Linux/macOS
```

## 最佳实践

### 1. 请求设计

- **场景时长**: 每个场景建议 5-15 秒，避免过长
- **文本内容**: 每屏文字不超过 50 字，保证可读性
- **动画数量**: 每个场景 2-4 个动画 cue，避免过度复杂
- **音频同步**: 确保 `durationSec` 与音频文件实际时长一致

### 2. 性能优化

- **分辨率选择**: 测试时使用 720p，生产环境使用 1080p
- **批量处理**: 大量任务时使用队列异步处理
- **资源清理**: 定期清理 `runtime/tasks` 中的旧任务
- **CDN 加速**: OSS 配置 CDN 提升视频加载速度

### 3. 错误处理

- **重试策略**: 利用内置的自动修复机制（最多 6 次）
- **降级方案**: LLM 不可用时切换到 Mock 模式
- **超时控制**: 设置合理的 `timeoutSec`（默认 600 秒）
- **日志记录**: 保留所有尝试记录便于问题排查

### 4. 安全建议

- **API 认证**: 生产环境添加 API Key 或 JWT 认证
- **输入验证**: 严格验证用户上传的音频 URL
- **速率限制**: 防止恶意请求耗尽资源
- **OSS 权限**: 使用 RAM STS 临时令牌而非永久 AK/SK

### 5. 监控告警

```python
# 集成 Prometheus 监控
from prometheus_fastapi_instrumentator import Instrumentator

Instrumentator().instrument(app).expose(app)
```

关键指标：
- 请求成功率
- 平均渲染时长
- 错误率分布
- 资源使用率
